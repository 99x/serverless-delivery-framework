Introduction

In software development process continuous delivery is important part that is most cost effective thing when we use in house servers. But when applying the tool and technique that exist for serverless application framework and platforms that is alternative for save cost. This application is a fundamental application for continous delivery with serverless architechture in Amazon Web Services (AWS)

Application Architechture

Seerverless Delivery Framework is just appliation of serverless platforms to deliver software versions to diffent users. Application users can use software diffent versions parallelly. Using this application system admin, developers can switch software versions for differnt end user. This framework can be used for any serverless framework related project.

The diagrame above shows the technology used to deliver software version for simple web applicatin. Let's lokk at what each componet provides

  • AWS C – All the techonlogies based on Amazon Web Services (AWS)
  • AWS  API Gateway – Each action in the pipeline can create a new artifact. The artifact becomes an output from the action in the pipeline and is stored in an S3 bucket to become an input for the next action.
  • AWS Lambda – You create Lambda functions to do the work of individual actions in the pipeline. For example, running a gulp task on a repository is handled by a Lambda function.
  • AWSS3 and Gulp – NPM is used for resolving all the dependencies of a given repository. Gulp is used for defining the tasks of the repository, such as running unit tests and packaging artifacts.
  • Testing tools – JSHint is used for performing static analysis of the code, Mocha is a unit test framework for JavaScript and Chai is a BDD assertion library for describing the unit tests.
  • AWS CloudFormation – CFN templates are used to create and update all these resources in a serverless stack.

Setup Application

1. Clone and Install Depandancies

$ git clone https://github.com/99xt/serverless-delivery-framework.git
          $ cd serverless-delivery-framework && npm install
          $ gulp dependancy-install

2. Create s3 Bucket and Configure

          $ gulp create-S3Bucket --name {YOUR-S3-Bucket-Name}
            $ gulp config-S3Bucket --name {YOUR-S3-Bucket-Name}

3. Serve Delivery Framework web client

$ gulp serve-client

Deploy Application

$ gulp deploy-webclient --version {Application-Version}
$ gulp deploy-authApp #
$ gulp deploy-api --stage {Stage-Name}
