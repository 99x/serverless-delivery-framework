Introduction

In software development process continuous delivery is important part that is most cost effective thing when we use in house servers. But when applying the tool and technique that exist for serverless application framework and platforms that is alternative for save cost. This application is a fundamental application for continous delivery with serverless architechture in Amazon Web Services (AWS)

Application Architechture

Seerverless Delivery Framework is just appliation of serverless platforms to deliver software versions to diffent users. Application users can use software diffent versions parallelly. Using this application system admin, developers can switch software versions for differnt end user. This framework can be used for any serverless framework related project.

The diagrame above shows the technology used to deliver software version for simple web application. Let's look at what each componet provides

  • AWS – All the techonlogies based on Amazon Web Services (AWS)
  •   API Gateway – When user request application URL. if user is unautherized it is redirect for authentication otherwise apigaway respond user relevent software version index page using indexpage lambda function
  • Index Page Lambda – You create Lambda functions to do the work of individual actions in the pipeline. For example, running a gulp task on a repository is handled by a Lambda function.
  • Authentication Application – Authenticate unautherized users this application can be any other service such as Auth0, Cognito etc. after sucessfull authentication redirect api gateway then api gateway relevent software versions for autherized user. of the repository, such as running unit tests and packaging artifacts.
  • Index Page Lambda Function- This lambda function dynamiclly generate application client index.html page
  • S3 Bucket - All the client applications are store in here. When deply new version this bucket authomatically create new folder and store all files in that folder.

Setup Application

1. Clone and Install Depandancies

Clone or download source from github

$ git clone https://github.com/99xt/serverless-delivery-framework.git
          $ cd serverless-delivery-framework && npm install
          $ gulp dependancy-install

Install all npm dependancies 

          $ cd serverless-delivery-framework && npm install
                $ gulp dependancy-install

2. Create s3 Bucket and Configure

Create simple storage service (S3) bucket for store client application source

          $ gulp create-S3Bucket --name {YOUR-S3-Bucket-Name}
            $ gulp config-S3Bucket --name {YOUR-S3-Bucket-Name}

3. Serve Delivery Framework web client

$ gulp serve-client

Deploy Application

$ gulp deploy-webclient --version {Application-Version}
$ gulp deploy-authApp #
$ gulp deploy-api --stage {Stage-Name}
